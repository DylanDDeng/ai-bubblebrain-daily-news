Boris Cherny 创造了 Claude Code，但他并没有对Claude Code做很复杂的使用配置。下面是他对于Claude Code 的使用方法： 

原推地址：<u>https://x.com/bcherny/status/2007179832300581177?s=20 </u>

1. 同时在终端里并行跑5个Claude Code 实例，并且打开系统通知，这样就可以知道Claude什么时候需要人为输入。  

![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/20260103112738946.png)

2. 同时在网页端跑5-10个Claude 实例，与电脑本地端也是并行跑，使用`--teleport` 命令来回切换。Boris 每天早上也会在移动端开启任务。  

![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/20260103112756135.png) 

3. 使用Opus 4.5 并且打开思考模式，这就是最好的写代码模型。 

4. 团队共同维护`CLAUDE.md`，如果Claude做错了什么事情，可以在`CLAUDE.md`中记录下来，避免下次犯同样的错。 

5. 代码审查阶段，在同事提的PR上@.claude,这样Claude会进行自动分析，并把内容添加到`CLAUDE.md`中。 这样产生知识复利效应。

6. 大部分的对话开启都来自规划模式，`Plan`模式真的很重要！ 

7. 多使用自定义的slash command，加速工作流程，避免每次相同的流程都要重复输入prompt。 Boris举了个例子，他会使用`/commit-push-pr` 这个slash command，来快速提交代码并创建PR。 

8. 使用子代理功能（subagents）。 Boris会使用`code-simplifier` 这个子代理，来简化Claude生成的代码，以及`verify-app`这个子代理来测试Claude完成的工作。 

9. 使用`PostToolUse`这个hook，来验证代码，保证不要出现格式上的错误，符合项目的代码规范。 

![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/20260103114216898.png) 

10. 不使用`--dangerously-skip-permissions`. 

11. Claude Code 配置MCP服务、工具等，包括Slack MCP， BigQuery， Sentry 等 

12. 对于长时间运行的任务，确保Claude能够独立自主的完成，并自我验证。Boris 写了几种方式，一种是通过Prompt 给子代理验证，还有一种是靠Hook 机制，更佳稳定，还有一种是可以通过ralph-wiggum 插件来验证。

13. 最核心的一点是给Claude建立自我验证的反馈循环，这能让结果质量提升2-3倍。
