大家好，

看到美团也发布了自家的AI IDE。   

国内真尼玛卷到起飞，

但我感觉美团更是卷中之卷，

俗称，**卷王** 
![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/shadow-image-1762772303951.png)

这一个多月，

我看着它们发了各种模型、论文、Benchmark 

甚至到现在连AI编程也加入进来。 
![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/shadow-image-1762771521912.png)   

它们的AI IDE 叫 **CatPaw**，

中文翻译为**猫爪**。 

再想到它们的模型，叫龙猫， 

整得还是挺有意义的。 

>下载链接在此： 
>https://catpaw.meituan.com/ 

唯一的问题是，<u>目前只支持Mac系统</u>，  

不过Windows 也快了，

官方说预计这周就会上Windows，  

![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/shadow-image-1762772576214.png)

Windows系统的小伙伴可以在耐心等等。  

先说说<u>整体感受</u>， 

还是非常类似Cursor的， 

基本上，如果你是非常熟悉Cursor的小伙伴，
可以做到无痛切换。  

主要Cursor现在20刀的用量真的不经用，

而且经常因为地区问题，

导致使用某些<u>“封号斗罗”</u>模型非常的麻烦。 

所以，

我心里还是挺期待看到国内公司想做出平替Cursor的产品的。

目前CatPaw**默认支持两个模式**，和Cursor非常类似，**Ask和Agent**。 

Ask就是只跟AI基于你的代码库进行问答聊天，不会自动帮你进行任何修改，除非你自己选择应用AI的修改。

而Agent就是AI会根据你的需求，自动帮你修改、运行代码。 

但是，在我使用的时候，

我发现了Ask模式有一个有点子<u>离谱</u>的问题！！

![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/shadow-image-1762774133747.png) 

Ask模式下它似乎根本读不懂这个项目在做什么。

对比看一下Agent模式下的正确回答： 

![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/shadow-image-1762774292873.png) 

而且这个Ask模式，

**我每次问这个项目在做什么的时候，它给的答案还都完全不一样。** 

真是离谱到家了，离谱他妈给离谱开门。。。
![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/shadow-image-1762774503352.png) 

我不知道是不是只有我一个人出现了这个bug。。。

所以，

**我先建议暂时别用这个Ask 模式。** 

除了默认支持的这两个模式之外，

CatPaw 还支持用户自定义Agent。 

![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/shadow-image-1762782484783.png)

这个确实让我觉得还蛮不错的，能做很多事。

你可以自己选择模型、设定系统提示词、选择MCP等等，作为你的专属Agent。 

模型方面，

目前CatPaw除了支持自家的`longcat-flash`， 

还支持`DeepSeek-V3.2`和`GLM-4.6`, 

当然，它还支持接入第三方的模型服务商，

主打一个内外兼修。 

还可以通过OpenRouter接入Claude模型。

![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/shadow-image-1762782907673.png)    

![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/shadow-image-1762783184675.png) 

我决定就用CatPaw的longcat-flash模型，给我的资料库做个新的功能。 

目前，它长这样： 

![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/shadow-image-1762783961180.png) 

好是挺好，

就是我希望能给我加个分页功能，

能以5个卡片作为一页。 

```
请帮我给prompt收录页面，目前有6个prompt卡片， 增加个分页功能，5个卡片为一页。
```

首先别的不说，

`longcat-flash`是真快啊， 

一想到Cursor2.0更新的Composer-1也是主打快。。。

怎么好像又找到了一个抛弃Cursor的理由。。。

![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/shadow-image-1762784765430.png) 

扯远了，

让我们来看看`longcat-flash`干的如何。 

![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/shadow-image-1762785172899.png) 

从读取文件，到代码修改都完成的不错，

整体流程和在Cursor里基本相似，

而且最后还会写个测试页面帮我验证功能是否完成。 

![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/shadow-image-1762785590553.png)

效果如下： 

![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/shadow-image-1762785708495.png)

样式和配色实现的还是挺统一的，

感觉像这类任务，用Codex CLI，Claude Code 因为速度的原因，反而影响了使用体验，

而`long-cat-flash`加持下的CatPaw体验就非常不错，

简直快得起飞！

![](https://image-1325800846.cos.ap-nanjing.myqcloud.com/shadow-image-1762788079637.png)
 
另外，现在新用户注册，享有500次积分。1个积分可以进行1次对话。 

## 写在最后  

从Claude Code 出来后，

我自己其实已经很少使用IDE来编程了， 

基本的编程工具就是Claude Code，Codex CLI 还有Kimi CLI 来回切换，

甚至连Cursor的会员都取消了订阅，

但并不是说IDE就真的没有市场， 

对于那些需要快速原型验证的场景，有个直观的界面还是香得一逼。

而且，`longcat-flash`的速度优势太明显了，

无论是Claude Code还是Codex CLI都没给我过这种超爽的体验。

从`longcat`到`CatPaw`，美团这是铁了心要把AI工具链打通。

再看看隔壁字节、阿里，恨不得把模型、IDE、CLI全给你包圆了，

但是不得不说，

这种卷，对我们用户来说是真的爽！
