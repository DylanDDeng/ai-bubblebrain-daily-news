## Production-only blockers

These require owner authority or real external infrastructure and cannot be satisfied by local code:

1. Complete the remaining Routine Access and attestation-key rotation drills.
2. Archive four real release Previews and the required production fault drills.
3. Obtain independent security, platform and Admin GO reviews with P0=0 and P1=0.
4. Complete Phase 5 cutover by committing the exact-artifact workflows and disabling the still-connected
   Cloudflare Git production publisher.
5. Complete the strict two-natural-day Asia/Shanghai observation window with all 8 real scheduled
   slots and no manual repair, rollback or unresolved P0/P1.
6. Deploy the section 9 observability workflow with least-privilege content credentials and archive
   a fired-and-recovered drill for batch freshness, manifest drift, API 5xx/cache and search freshness.

ADR-003 records the owner's explicit authorization to activate production with these residual items
tracked as follow-up evidence. The unique-publisher cutover, runtime preflight and smoke verification
remain activation requirements.
