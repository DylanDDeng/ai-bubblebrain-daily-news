## Design Context

### Users

The primary user is the owner of this personal knowledge base, with public readers as a secondary
audience. They visit on desktop and mobile to scan a high-volume AI news day quickly, understand what
happened in chronological order, filter by content type or topic, search titles and summaries, and
open the original source without fighting long URLs or dense legacy Markdown. The page must remain
useful as server-rendered HTML when JavaScript is unavailable.

### Brand Personality

Calm, editorial, precise. The interface should feel like a well-edited research desk: trustworthy,
quietly technical, and fast to scan. It should reduce information anxiety rather than manufacture
urgency or look like an AI demo.

### Aesthetic Direction

Use the user-provided AIHOT timeline reference as the interaction and information-density anchor:
a slim chronological rail, clear time labels, compact source metadata, strong titles, restrained
summaries, and practical filters. Translate that reference into Bubble's Brain rather than cloning
its brand. Favor a light, cool-tinted paper surface with deep ink and a sparse teal accent; support a
carefully tuned dark theme for the existing site. Avoid cyberpunk neon, generic purple gradients,
glassmorphism, oversized hero metrics, nested cards, and decorative AI imagery.

Typography should be optimized for mixed Chinese and Latin editorial reading, with tabular numerals
for time. Layouts should be content-dense but not cramped, using rules, alignment, and spacing before
shadows. Motion is limited to short state transitions and must respect reduced-motion preferences.
Interactive controls require visible focus states, 44px touch targets, and WCAG AA contrast.

### Design Principles

1. Chronology is the spine: time, batch, and date navigation stay visually stable while content varies.
2. Progressive disclosure beats omission: show title/source/time first, summary next, advanced detail on demand.
3. Search and filters clarify the corpus and always produce shareable URL state; they never hide the unfiltered HTML baseline.
4. Preserve reading trust: source links are explicit, external, keyboard accessible, and long URLs never become visible layout content.
5. One data contract, two renderers: Hugo and Astro must express the same information hierarchy and route behavior.
